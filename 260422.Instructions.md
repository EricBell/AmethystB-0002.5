
Objective
In this project, you will create professional tasks that Gemini CLI fails at without a skill file but succeeds at with a well-written skill file.

What is a skill file? A skill file is a document you write (in Markdown format, i.e. a .md file) that gives Gemini CLI background knowledge and instructions for how to approach a specific type of task. It tells Gemini what knowledge to have, what methods to use, what mistakes to avoid, what domain conventions to apply, and so on. Think of it like training a new team member — you're teaching the approach and methodology, rather than giving them the answers to a specific task. Gemini CLI reads the skill file before it starts working, so it can apply that guidance to its response.

Task description examples
Example:

I will provide an Excel workbook containing 5 years of borrower financial data, along with notes on accounting treatment and non-recurring items. I then will ask the model to analyze the historical performance, determine reported vs. adjusted EBITDA, calculate historical and pro forma credit metrics, assess whether a proposed incremental debt issuance would comply with a 4.5x Debt/EBITDA covenant, run a downside stress case, identify key credit risks, and make a recommendation to an Internal Credit Committee. This task requires genuine expertise because the dataset contains several traps that only a credit research analyst would navigate well - things like recognizing that impairments reported on a separate line from D&A need to be folded into reported EBITDA, or knowing that a litigation settlement is counted as cash received (so it stays in FCF) while a gain on asset sale is a non-cash accounting entry (so it gets stripped out).

Example:

I will provide a spreadsheet of groundwater monitoring data from six wells across three annual sampling events at a site in Ontario, Canada. The data includes BTEX compounds, metals, and general chemistry parameters. This task requires genuine expertise because the dataset contains several domain-specific traps — like knowing that interim PWQOs take precedence over permanent objectives under Guideline 1.10, that a value which rises then falls across three sampling events is not an "increasing trend," and that screening-level findings should never be characterized as compliance violations or attributed to specific contamination sources.

Step 3. Develop Your Skill File
This is the most important step. Your skill file must be high-quality, generalizable instructions — clear and thorough enough that it could serve as a professional reference guide for this type of work.

Where to place your skill file:
Your skill file needs to go in a specific location inside your project folder. Here’s how to set it up:

Pick a short name for your skill. This should describe what your skill does, using all lowercase letters with hyphens between words. For example: nda-clause-editor, invoice-reconciler, or soap-note-formatter.

Create the folder structure. Inside your project folder, you need to create the following nested folders:

your-project-folder/
  .gemini/
    skills/
      your-skill-name/
        SKILL.md    ← your skill file goes here
How to create these folders:

Mac/Linux: In your terminal (make sure you’re in your project folder), run:

mkdir -p .gemini/skills/your-skill-name
(Replace your-skill-name with the name you chose in step 1.)

Windows (PowerShell): Run:

New-Item -ItemType Directory -Path ".gemini\skills\your-skill-name" -Force
Or use your file explorer — just create the folders manually one at a time. Note that .gemini is a hidden folder (starts with a dot), so you may need to show hidden files to see it after creating it. See the Reference Sheet for how to show hidden files on your system.

Create the SKILL.md file inside the skill-name folder. You can use any text editor to create this file. Save it with the exact name SKILL.md (all caps, with the .md extension).

Important: The skill name you chose in step 1 must match the name: value in the header of your SKILL.md file (explained below). Your skill folder should contain only the single SKILL.md file. Use a new, empty project folder for each task — leftover files or skills from prior tasks can contaminate your run.

How to develop your skill file:
Every SKILL.md file needs to start with a special header block that tells Gemini CLI what the skill is for. The header looks like this:

---
name: your-skill-name
description: Use this skill when the user asks to [describe the type of task this skill helps with].
---
A few things to note:

The name must exactly match the folder name you created above.

The description is how Gemini decides whether to use your skill — write it as a clear sentence starting with “Use this skill when...” that describes the type of task (not the specific task you’re testing).

The --- lines above and below are required. Make sure there’s a blank line after the closing --- before the rest of your content.

Example header:

---
name: credit-analysis
description: Perform credit analysis on a borrower's financial data. Use this skill whenever the user provides a financial dataset and asks for leverage analysis, covenant compliance testing, capital structure modeling, or stress testing. Also trigger when the user mentions terms like 'debt issuance', 'covenant', 'leverage ratio', 'interest coverage', or 'credit risk assessment'. The skill produces two deliverables: (1) a credit analysis memo (in a standard text format like .docx) and (2) a supporting financial model with formulas (in a standard spreadsheet format like .xlsx).
---
For the body of the skill you will:

Diagnose the severe failures: Review Gemini's Run 1 output and identify exactly where and why it went critically wrong.

Identify the root causes: Was it missing domain context? Wrong methodology? Ignored conventions? Bad output format?

Write instructions that address those failures — but do it in a generalizable way. The instructions should help with this type of task, not just this specific instance.

After saving your SKILL.md file, you need to tell Gemini to pick it up. Either close and reopen Gemini CLI, or type /skills reload in the Gemini CLI session.

Type /skills list in Gemini CLI to check that your skill shows up. If it doesn’t appear, double-check:

Both --- lines in your header are on their own lines (nothing else on those lines)

There’s a blank line after the second --- before the rest of your content

The name: in your header exactly matches your folder name

When you paste your task prompt in Run 2, Gemini should ask you for permission to use your skill. Say yes (approve it). If Gemini doesn’t ask about your skill, it means Gemini didn’t recognize your prompt as matching your skill’s description. To fix this:

Make your skill’s description: line more closely match the kind of task your prompt describes

Reload the skill by running /skills reload or restarting Gemini CLI

Remember to use /clear and a fresh copy of your input files when retrying

Critical requirements:
The skill file should use Markdown format (click here for a guide).

DO NOT include specific answers. The skill file should teach the approach, methodology, and standards — not give away the expected output.

Be specific and actionable. "Follow best practices" is unhelpful. "Use FIFO inventory valuation and round all currency values to two decimal places" is useful.

Include domain-specific conventions — citation formats, calculation methods, terminology, regulatory requirements, etc.

Specify output format requirements — file types, naming conventions, structure, organization.

Structure clearly with headers, sections, and hierarchy so the AI can easily parse the instructions. Use the example skill provided as an example of the structure needed.

Aim for 500–2,000 words. A good skill file is long enough to cover the key rules, traps, and methodology for your domain, but not so long that it becomes padded with filler. Most strong skill files land in this range.

The skill file should apply to other tasks similar to yours; don't tailor the skill file just to your exact task.

For example, if your task asked for an xlsx file (but a similar task might have validly asked for a csv file), you shouldn't tell the skill file to only use an xlsx file. In this case, you could write something in the skill file like "Output a spreadsheet in a standard format (for example, .xlsx or .csv)."

Or if there are two valid methods to calculate a certain value in your field, then your skill should allow the model to use either method.

Remember, the model will still see any specific instructions in your prompt. The skill file is like a general training that helps the model complete tasks like yours.

# Credit Analysis Skill

## Workflow


1. Load and Map the Dataset

Read the input data file. Identify and map every metric to one of these categories: top-line performance (Revenue), operating earnings (EBIT), non-cash charges (D&A, Impairment), non-recurring items (Other_Income, Restructuring, Gain_on_Sale, Litigation), debt balances, interest expense, taxes, working capital changes, and capital expenditures. If column names are ambiguous, verify against the notes sheet — never assume a column's role from its name alone.


2. Interpret the Notes

Read every note in the notes sheet. For each note, determine which year(s) and line item(s) it applies to, and whether it indicates a non-recurring, non-operating, or non-cash event. Maintain a ledger of adjustments by year. Only flag items as adjustments when the notes explicitly describe them as non-recurring or not reflective of ongoing performance.


3. Determine if Impairments are included in D&A

If the dataset has a **separate `Impairment` column** distinct from `D&A`, treat D&A as **excluding** impairment — a separately reported line item is, by standard accounting convention, not already embedded in another line item on the same sheet. Cross-check against the notes; notes trump structural inference.

Only if no separate impairment column exists should you fall back on heuristics (e.g., an unusually large D&A figure in a year with a known write-down). Note that EBIT as reported typically already contains Other_Income and non-recurring gains/losses above the EBIT line — these do not need to be added back when deriving Reported EBITDA; they are handled in the Adjusted EBITDA step.


4. EBITDA Calculation

If D&A does not include impairments: Reported EBITDA = EBIT (as reported) + Depreciation & Amortization + Impairments
If D&A does include impairments: Reported EBITDA = EBIT (as reported) + Depreciation & Amortization


5. Adjusted EBITDA Calculation

Apply adjustments only where the notes identify non-recurring items:

- **Subtract** non-recurring gains (one-time income, asset disposal gains, litigation settlements received, restructuring benefits).
- **Do not add back** non-recurring charges (impairment write-downs, restructuring costs, one-time litigation expenses), unless specifically told to.
- **Add back** non-cash stock-based compensation if material and if the credit agreement defines EBITDA to exclude it.

Adjusted EBITDA = Reported EBITDA − Non-recurring gains + Non-recurring charges (only if specifically told to add back) + Non-cash add-backs

For years where no non-recurring items are identified, Adjusted EBITDA equals Reported EBITDA.


6. Leverage and Coverage Ratios

Calculate using **Adjusted EBITDA** as the denominator:

| Metric | Formula |
|--------|---------|
| Debt / EBITDA (Leverage) | Total Debt ÷ Adjusted EBITDA |
| Interest Coverage Ratio | Adjusted EBITDA ÷ Interest Expense |

Use Total Debt (all tranches) in the numerator, not just senior debt, unless the covenant definition specifies otherwise.


7. Free Cash Flow after Capital Expenditures

FCF after CapEx = Reported EBIT − Cash Taxes − Cash Interest Expense − non-recurring non-cash gains + D&A + Impairment + Net Working Capital Change − Capital Expenditures

**Cash vs. non-cash gain treatment — critical:** Only *non-cash* gains should be stripped out of FCF. A cash gain (e.g., a litigation settlement actually received) represents real cash and must remain in FCF. A non-cash accounting gain (e.g., "gain on sale of an asset" — the book-value-vs-proceeds difference) must be subtracted out. When the notes describe an `Other_Income` line item, determine which category it falls into before deciding whether to subtract it.

**Working capital sign convention:** Two common conventions exist:
- **Changes-in-cash-flow convention** (most common): Negative = cash used, positive = cash generated. Sum directly.
- **Balance-sheet-delta convention**: Increases in current assets are cash outflows (subtract); increases in current liabilities are cash inflows (add).

Inspect the data to determine which convention is in use.


8. Pro Forma Capital Structure

Using the **most recent fiscal year** as the base:

Pro Forma Total Debt       = Existing Total Debt + New Debt Principal
Pro Forma Interest Expense = Existing Interest Expense + (New Debt Principal × Total Interest Rate)
Pro Forma Debt / EBITDA    = Pro Forma Total Debt ÷ Adjusted EBITDA
Pro Forma Interest Coverage = Adjusted EBITDA ÷ Pro Forma Interest Expense


9. Financial Covenant Compliance

Test pro forma leverage against every financial maintenance covenant. If breached, calculate maximum additional debt:
   Maximum Total Debt at Covenant Limit = Adjusted EBITDA × Covenant Threshold
   Maximum Additional Debt = Maximum Total Debt at Covenant Limit − Existing Total Debt

If not breached, calculate headroom in turns of EBITDA and dollar terms.


10. Stress Testing

Apply the downside scenario specified in the prompt (or default to 10%–15% EBITDA decline):

Stressed EBITDA            = Adjusted EBITDA × (1 − Decline %)
Stressed Debt / EBITDA     = Pro Forma Total Debt ÷ Stressed EBITDA
Stressed Interest Coverage = Stressed EBITDA ÷ Pro Forma Interest Expense

Address whether the borrower breaches the covenant under stress, how much additional erosion it could absorb, and implications for refinancing risk.


11. Recommendation

Formulate a clear recommendation: **Support** (comfortably within covenants under stress), **Support with Conditions** (within covenants but limited headroom — specify conditions such as reduced size, tighter covenants, mandatory hedging), or **Decline** (covenant breach, insufficient cash flow). The recommendation must cite specific leverage metrics, covenant compliance, and stress scenario results.


## Output format

Produce two deliverables: (1) a credit analysis memo (.docx) covering executive summary, historical performance with EBITDA adjustments explained, proposed transaction, pro forma analysis, stress scenario, and recommendation; and (2) a supporting financial model (.xlsx) with formulas in all calculated cells (never hardcoded values), organized into a Historical Metrics sheet and a Pro Forma Analysis sheet with cross-sheet references.

borrower_data.xlsx

# Groundwater Quality Assessment Skill

## Workflow

### Step 1: Inventory the data

Catalogue every monitoring well (noting aquifer level — e.g., "A" = upper, "B" = lower), every sampling event by date, and every analyte by category (BTEX, VOCs, metals, general chemistry). Note units (typically µg/L for organics/trace metals, mg/L for major ions/alkalinity). Identify non-detects and their reporting limits — non-detects are not the same as zero and must never be treated as zero in comparisons.

---

### Step 2: Apply the correct regulatory guidelines (PWQOs)

For Ontario sites, the benchmark is the **Ontario Provincial Water Quality Objectives (PWQOs)**. Key rules:

**Rule 2.1 — Interim PWQOs take precedence (Guideline 1.10).**
Where an analyte has both a permanent and an interim objective, use the **interim objective**. This is a common error — do not default to the permanent objective when an interim exists. Examples: arsenic, lead, vanadium, m-xylene all have interim PWQOs.

**Rule 2.2 — Total Xylenes must be assessed using the m-Xylene interim PWQO.**
There is no "Total Xylenes" PWQO. Assess Total Xylenes against the most restrictive isomer objective: the interim PWQO for m-xylene. Do not use o-xylene or p-xylene objectives, and do not leave Total Xylenes unevaluated.

**Rule 2.3 — Alkalinity cannot be evaluated against a PWQO.**
Alkalinity's appropriate benchmark is the natural background concentration for the local geology, not a fixed provincial objective. Report the values but note that no PWQO applies. Do not fabricate or substitute a guideline.

**Rule 2.4 — Compile a PWQO reference table** listing each analyte, its PWQO value, objective type (permanent/interim), and any notes. This table becomes part of the deliverable.

---

### Step 3: Identify exceedances

An exceedance occurs when a detected concentration exceeds the applicable PWQO. Key rules:

- **Non-detects are not exceedances.** A result of "<0.5 µg/L" where the PWQO is 0.8 µg/L is not an exceedance.
- Compare the reported value as-is — do not discard or adjust before comparison.
- Use the correct PWQO per Step 2 rules (especially interim objectives and the m-xylene proxy for Total Xylenes).
- Note which wells and aquifer zones show exceedances. If exceedances occur only in one aquifer, state this explicitly.
- **Do not characterize exceedances as compliance failures.** A screening-level assessment identifies exceedances relative to guidelines, not compliance status.

Create an Exceedance Summary Table (Well ID, Aquifer, Analyte, Date, Result, PWQO, Exceedance Factor).

---

### Step 4: Handle anomalous values

**Rule 4.1 — Report as detected; do not remove or correct.** Flag in the data table and note in text.

**Rule 4.2 — Describe factually.** State the value, well, date, and magnitude of departure. Example: "Vanadium at 101 µg/L in MW-02B (2025), compared to 0.91 and 0.84 µg/L in other years."

**Rule 4.3 — Do not speculate on causes.** Use neutral language: "the cause has not been determined" or "further investigation may be warranted."

**Rule 4.4 — Still compare against PWQOs.** An anomalous value that exceeds a PWQO is an exceedance. Include it in the summary table.

---

### Step 5: Characterize temporal trends

Where three or more sampling events are available, apply these rules strictly:

**Rule 5.1 — A trend requires consistent directional change across ALL events.** A value that rises from 2024 to 2025 but falls from 2025 to 2026 is NOT increasing — it is variable or non-monotonic. "Increasing" means it increases at every successive event. "Decreasing" means it decreases at every successive event.

**Rule 5.2 — Do not call a non-monotonic pattern a trend.** If values rise then fall, describe accurately: "concentrations increased between 2024 and 2025 before declining in 2026."

**Rule 5.3 — Anomalous values disrupt trend characterization.** Do not use an anomalous data point to assert a trend.

**Rule 5.4 — Never speculate on why a trend exists.** Describing a pattern is appropriate; attributing it to a source or activity is not.

---

### Step 6: Additional analytical requirements

- **Spatial distribution:** Note whether exceedances are limited to one aquifer or present in both, and whether all or only a subset of wells in the affected aquifer show exceedances. Do not attribute spatial patterns to contamination sources.
- **Temporal focus:** The most recent sampling event is the primary focus — current conditions drive decision-making. Earlier events provide context.
- **Tone:** Use precise numerical values, neutral language, and past tense for historical data / present tense for current conditions. Never assert a cause for detected concentrations. Never make definitive compliance, liability, or remediation statements.

---

### Step 7: Common domain traps — summary reference

| Trap | Incorrect Approach | Correct Approach |
|---|---|---|
| Interim vs. permanent PWQO | Use permanent PWQO when an interim also exists | Use interim PWQO per Guideline 1.10 |
| Total Xylenes comparison | Leave unevaluated or use wrong isomer objective | Compare against m-xylene interim PWQO |
| Alkalinity assessment | Compare against a surrogate or invented guideline | Report values; note no PWQO applies (natural background not defined) |
| Anomalous values | Remove, average out, or attribute to error | Report as-is; describe factually; compare against PWQO; note in text |
| Trend characterization | Call any two-point increase a trend | Require consistent directional change across all available events |
| Speculating on causes | "Contamination from the former fuel station..." | "Elevated concentrations were detected in..." |
| Regulatory conclusions | "The site is non-compliant and requires remediation" | "Concentrations exceed PWQOs in [well/aquifer] as of [date]" |
| Aquifer attribution | Aggregate all wells into one summary | Separately characterize upper and lower aquifer wells |
| Non-detects as exceedances | Flag "<RL" as an exceedance if RL > PWQO | Non-detect is not an exceedance; note the elevated reporting limit |
| Primary temporal focus | Treat earliest data as the main findings | Most recent sampling event is primary; earlier events are context |

---

## Output format

The deliverable is a professional technical memo with numbered sections (Purpose/Scope, Site Background, Methodology, PWQO Reference Table, Groundwater Quality Data, Exceedance Summary, Temporal Trends, Spatial Distribution, Limitations, Conclusions). Include all data tables inline or as appendices. State limitations explicitly (limited sampling frequency, seasonal gaps, screening-level scope, analytes without applicable guidelines, anomalous data influence, well network coverage).

Example 3: IP Litigation Board Briefing — Interim Injunction Analysis (Law)
Prompt:
Act as the General Counsel of Blue Horizon Theatricals Ltd. Using the attached documents relating to the touring musical “Sunset Nights: A Pop Musical Celebration” operated by Brightstar Touring Ltd — which may infringe on the company’s established show “Summer Nights: the Musical” — prepare a confidential board briefing of 1000 words (±10%) recommending whether to: (1) seek an urgent interim injunction, (2) issue pre-action correspondence, (3) pursue a commercial settlement, or (4) take no action yet.

Your analysis must separately address each IP right: copyright in the dramatic work, copyright in musical compositions, trademark infringement, and passing off. Explain how publisher and artist alignment affects claim strength.

If an interim injunction is considered, apply the American Cyanamid principles under the laws of England and Wales in the correct order. Consider adequacy of damages from both perspectives, and analyze the balance of convenience separately. Explain the purpose and practical risk of the cross-undertaking in damages.

Address the February 2026 email from the junior commercial exec: whether they had actual authority, whether apparent authority or estoppel might be argued, and what BrightStar would need to establish for estoppel to succeed. Consider BrightStar’s solvency and the delay between first awareness and formal escalation.

Identify at least four material weaknesses with the legal consequence of each. Provide a main recommendation and a fallback strategy, and explain what would most likely cause your preferred strategy to fail.

Input files:
sunset_nights_touring_agreement.pdf, theatrical_production_document.pdf, production_contract_terms.docx, performance_financial_report.pdf, tour_schedule_document.pdf

Expected output:
A fully correct output should consist of a confidential board briefing.

Each IP right must be analyzed separately — copyright in the dramatic work, copyright in musical compositions, trademark infringement, and passing off — never merged together. The passing off analysis must explicitly apply the three-part test from Reckitt & Colman v Borden: goodwill, misrepresentation, and damage, each as a distinct element.

# IP Litigation Board Briefing Skill

## Workflow

### Step 1 — Identify and Separately List Every IP Right in Play

Catalogue every distinct IP right that may be infringed. For theatrical or creative works this typically includes: copyright in the dramatic work, copyright in musical compositions (each song separately if different authors), copyright in sound recordings/performances, registered trade marks, and unregistered trade marks / passing off.

**Critical rule:** Each right must be analyzed in its own section. Do not merge copyright with passing off, and do not merge different copyright categories (dramatic vs. musical). Each has different elements, ownership chains, defences, and remedies.

---

### Step 2 — Confirm Ownership and Standing

For each right, confirm: who created the work (employee vs. contractor — if contractor, was there a written assignment?), whether assigned or licensed (exclusive vs. non-exclusive), whether co-owners exist, and whether the claimant has standing.

**For musical compositions:** Confirm with the composer/publisher and the performing artist (if different) whether any license was granted to the alleged infringer. Alignment that no license was granted materially strengthens the claim.

**Critical rule:** An exclusive licensee under s.67 CDPA 1988 has standing to bring proceedings but may need to join the copyright owner.

---

### Step 3 — Analyze Each Copyright Claim Separately

For each copyright work, address these elements:

**3a. Subsistence:** Confirm originality (*Infopaq* [2009]), qualifying work status, and that copyright subsists (life + 70 years under s.12 CDPA 1988).

**3b. Scope of protection — the idea/expression dichotomy:** Identify which elements are protected expression vs. unprotectable ideas, styles, or genre conventions. This distinction is critical for dramatic works where themes and structure overlap with genre norms and commonly limits claims in theatrical and musical contexts.

**3c. Copying:** Assess access to the claimant's work and objective similarity in protectable expression. Both subconscious and deliberate copying can infringe.

**3d. Substantial part:** The test is qualitative not quantitative (s.16(3) CDPA 1988). The quality and importance of the copied portion within the original is the key question.

**3e. Defences:** Fair dealing (s.30, s.30A CDPA), incidental inclusion (s.31), and implied licence from course of dealing or industry practice.

**3f. Strength assessment:** Conclude each section with a frank assessment — strong, moderate, or weak — with brief reasons.

---

### Step 4 — Analyze Trademark Infringement (if applicable)

Apply the Trade Marks Act 1994. Confirm registration validity and specification. Apply the relevant ground: s.10(1) (identical mark/goods — no confusion required), s.10(2) (similar mark/goods + likelihood of confusion — conduct global assessment of visual, aural, conceptual similarity), or s.10(3) (mark with reputation + dilution). Consider honest concurrent use (s.11(3)) and acquiescence (s.48).

---

### Step 5 — Analyze Passing Off Under the Classic Three-Part Test

Apply the test from *Reckitt & Colman v Borden* [1990] (the "Jif Lemon" case). Each part must appear as a distinct sub-section.

**Part 1 — Goodwill:** Demonstrate subsisting goodwill in a business carried on in England and Wales. Goodwill attaches to the trading activity, not the work in isolation. Quantify: years of trading, audience size, revenue, awards, recognition. Assess honestly — acknowledge if limited in scope.

**Part 2 — Misrepresentation:** The defendant's conduct must make a material misrepresentation to the relevant public (the likely audience) that its goods/services are associated with the claimant. Assess overlap of audiences, similarity of name/get-up/marketing, and whether a substantial proportion would be deceived.

**Part 3 — Damage:** Show actual damage or real risk of damage to goodwill: diversion of trade, lost licensing opportunities, dilution of distinctiveness, reputational damage.

**Critical rule:** All three parts must be established independently. Weakness in any single part defeats the claim entirely. Disclose this to the board as a material weakness if it applies.

---

### Step 6 — Apply the American Cyanamid Test in the Correct Order

Apply *American Cyanamid Co v Ethicon Ltd* [1975] **in strict sequence**. Getting the order wrong is a fundamental analytical error.

**Stage (a) — Serious issue to be tried:** Is there a real (not fanciful) prospect of success? This is a low threshold. Assess each cause of action separately.

**Stage (b) — Adequacy of damages (two limbs, both required):**

*Limb (i) — Claimant:* Would damages adequately compensate the claimant if the injunction is refused but it succeeds at trial? Factors pointing to inadequacy: irreparable reputational harm, difficulty quantifying lost goodwill, loss of unique market timing.

*Limb (ii) — Defendant:* If the injunction is granted but the defendant succeeds, would the cross-undertaking adequately compensate the defendant? Assess defendant solvency here.

**Stage (c) — Balance of convenience:** Weigh: relative scale of harm; status quo (courts resist newcomers entrenching a new status quo); merits only as a tiebreaker; conduct of the parties including delay.

**Critical rule on delay:** Delay weakens urgency but does not automatically defeat relief. Frame any delay precisely — investigation, expert analysis, legal advice. Every week of further inaction weakens urgency arguments.

---

### Step 7 — Cross-Undertaking in Damages

Explain for a non-lawyer board audience:

**Purpose:** The claimant promises to compensate the defendant if the injunction is granted but the claimant ultimately loses. This is the court's primary mechanism to protect defendants.

**Practical risk:** If the defendant suffers losses while the injunction is in force and the claimant loses, the cross-undertaking exposes the claimant to a significant damages inquiry. The board must understand this as a real financial liability, not a formality.

---

### Step 8 — Authority and Estoppel Analysis

Where an allegation arises that a company representative previously granted permission to the defendant:

**Actual authority:** Assess express or implied actual authority. Granting IP licenses is a significant legal act — a junior executive will not ordinarily have implied authority without specific delegation.

**Apparent authority:** The company may be bound if it represented to the third party that the individual had authority (*Freeman & Lockyer v Buckhurst Park Properties* [1964]). Critical rule: the representation must originate from someone with actual authority — an agent cannot represent their own authority.

**Estoppel:** The defendant must establish all three elements independently: (1) clear and unequivocal representation, (2) reliance — the defendant acted differently because of it, (3) detriment — the defendant is worse off as a result. A representation alone is insufficient without reliance and detriment.

---

### Step 9 — Identify Material Weaknesses

Identify a minimum of four material weaknesses. For each, state: what it is, the legal consequence, and possible mitigation steps. Key categories:

- **Idea/expression dichotomy:** Copyright may not protect broad themes or genre conventions — claim narrowed or defeated
- **Narrow goodwill:** Limited trading in England/Wales defeats passing off at the first hurdle
- **Delay:** Weakens urgency for interim relief and may reduce recoverable damages
- **Authority gap / estoppel risk:** Contested permission defence increases litigation risk
- **Solvency of claimant:** If unable to fortify cross-undertaking, interim relief may be unavailable

---

### Step 10 — Defendant Solvency (Dual Context)

Address in two separate contexts:

**Context 1 — Adequacy of damages (Stage b):** A financially precarious defendant means damages cannot be enforced — supports interim relief.

**Context 2 — Balance of convenience (Stage c):** An injunction that destroys a fragile defendant may cause disproportionate harm — courts may be reluctant to grant relief unless the claimant's case is very strong.

Both contexts must appear as distinct discussions.

---

### Step 11 — Strategic Options and Recommendation

Present options with pros and cons: (A) seek interim injunction, (B) pre-action correspondence, (C) commercial settlement/licensing, (D) no immediate action.

**Primary recommendation:** State clearly which option is recommended, with reasons anchored in the legal analysis. Avoid hedging to the point of giving no real guidance.

**Fallback strategy:** What action if the primary recommendation fails.

**Critical failure point:** The single factor most likely to cause the strategy to fail, and what can be done now to reduce that risk.

---

## Output format

Sections in order: (1) Cover/header with CONFIDENTIAL — LEGALLY PRIVILEGED marking; (2) Executive summary with headline recommendation; (3) Background and factual matrix; (4) IP rights analysis — one section per right (copyright dramatic, copyright musical, trademark, passing off with three-part test); (5) Interim injunctive relief (American Cyanamid in correct sequence); (6) Cross-undertaking; (7) Authority/estoppel analysis; (8) Material weaknesses (minimum four); (9) Defendant solvency (dual context); (10) Strategic options; (11) Recommendation with fallback and critical failure point. Written in formal legal English, self-contained for non-lawyer board members.
